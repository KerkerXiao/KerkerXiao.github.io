reset

git reset 是一个功能强大的 Git 命令，用于回退版本和更改提交历史。它可以影响工作区、暂存区和提交历史，具体取决于你使用的选项。

其中，<mode> 可以是以下几种：

  • --soft：仅重置到指定提交，并保留所有更改在暂存区和工作区。
  • --mixed（默认）：重置到指定提交，并将更改从暂存区移除，但保留在工作区。
  • --hard：重置到指定提交，并将更改从暂存区和工作区全部移除。

本地仓库工作区域

  • 工作区 (Working Directory)：.git所在目录

工作区是你实际进行开发和编辑代码的地方。当你克隆一个 Git 仓库时，所有的文件都会被复制到你的工作区。工作区是包含了你项目所有文件的文件夹。

  • 暂存区 (Staging Area)：.git/index所在目录

暂存区是一个临时区域，用于记录将包含在下次提交中的文件快照。你可以选择性地将文件的某些更改添加到暂存区，从而控制提交的粒度和内容。

  • 本地仓库 (Local Repository)：：.git/objects所在目录

本地仓库是一个隐藏的 .git 目录，其中包含了所有的 Git 元数据和对象库。它保存了项目的所有历史版本，是 Git 实现版本控制的核心。

三者之间的关系

  1. 工作区到暂存区： 当你编辑文件并准备提交时，你需要先将文件添加到暂存区。这一步是使用 git add 命令完成的。
  2. 暂存区到本地仓库： 在将文件添加到暂存区后，你可以使用 git commit 命令将这些文件快照提交到本地仓库中。 git commit 只会提交暂存区的文件，不会提交工作区文件。
  3. 本地仓库到远程仓库： 在本地仓库中提交更改后，你可以使用 git push 命令将更改推送到远程仓库。

文件状态

在 git status 中，文件的状态可以是以下几种：

  1. Untracked files：这些文件没有被 Git 追踪。它们在工作区中，但还没有被添加到 Git 仓库。
  2. Tracked files：
    • Unmodified：文件被 Git 追踪，但自上次提交以来没有修改。
    • Modified：文件被 Git 追踪，并且自上次提交以来已经修改，但还没有添加到暂存区。
    • Staged：文件被 Git 追踪，修改后的内容已经被添加到暂存区，等待提交。
